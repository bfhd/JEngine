12.2.2 PPR/ACK - patient problem message (events PC1, PC2, PC3)

The patient problem message message is used to send problems from one application to another (e.g., a point of care system to a clinical repository). Many of the segments associated with this event are optional. This optionality allows systems in need of this information to set up transactions that fulfill their requirements.
